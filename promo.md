# 🎙️ VTT Keyboard - 你的桌面级智能语音输入与 AI 助理

VTT Keyboard 是一款基于 Rust (Tauri) 与 React 构建的跨平台桌面语音打字助手。它不仅仅是一个“语音转文字”工具，更是一个将**语音输入**、**系统全局操控**与**大语言模型 (LLM)** 深度结合的桌面效率神器。

无论是程序员、内容创作者，还是重度文字工作者，VTT Keyboard 都能让你真正解放双手，用嘴“敲”代码、写文档。

---

## ✨ 核心功能亮点 (Core Features)

### 1. ⌨️ 全局一键呼出，即说即敲 (Global Voice Input)
无需来回切换窗口！在任意应用程序（如 Word、微信、VS Code 等）中，按下你自定义的全局快捷键即可开始录音，松开快捷键瞬间完成识别，文本会**自动键入**到你当前的光标位置，全程丝滑无缝。

### 2. 🧠 多维引擎，随心切换 (Multi-Engine Support)
VTT Keyboard 内置了对多种业界顶级语音识别引擎的支持：
- **OpenAI Whisper**：提供极高准确率的语音识别体验。
- **火山引擎 (Volcengine)**：来自字节跳动的商业级语音服务，支持流式极速响应。
- **SenseVoice (本地私有化部署)**：完全免费、断网可用，极速且隐私绝对安全。

### 3. 🤖 AI 智能触发卡片 (Smart Trigger Cards) - *杀手级特性！*
这不仅仅是打字机，而是你的**语音指令助理**。
你可以预先定义各种“触发词”与对应的大模型 Prompt。例如：
- 设置关键词：`翻译为{value}`
- 实际场景：按住快捷键说“**翻译为英文：今天天气真好。**”
- 魔法发生：VTT 会自动拦截文本，剥离触发词，唤醒内置的大模型能力，最终光标处自动输入的是：*“The weather is really nice today.”*
完美实现**“语音指令 + AI 处理 + 自动上屏”**的闭环！

## 💡 设计与技术巧思 (Design & Tech Ingenuity)

在开发 VTT Keyboard 时，我们在架构和用户体验上倾注了大量的心血：

### 🛠️ 1. Tauri + Rust 的极致轻量化与极速体验
市面上大量桌面应用采用臃肿的 Electron，而我们选择了 **Tauri + Rust** 架构。VTT Keyboard 运行时常驻系统托盘，内存占用极低。在保持现代化 React UI 界面（如无边框极简窗口）的同时，彻底摆脱了系统资源枯竭的烦恼。对于一个随时待命的后台辅助工具来说，这一点至关重要。

### 🖼️ 2. 无感原生状态悬浮窗 (Native Status Overlay)
语音打字最怕“不知道当前是在录音还是在识别”。
我们在 Rust 层（`src-tauri/src/status_native.rs`）调用了操作系统原生图形 API（Windows Win32+GDI / macOS Cocoa / Linux GTK3），在屏幕绘制了一个极简轻量、悬浮且不抢夺光标焦点的状态指示器（显示：正在录音、识别中、已完成、错误）。
**巧思在于**：绕过了 WebView 的渲染延迟和焦点抢占，做到了真正的“无感提示”，你的打字思路绝不会被打断。

### 🐳 3. Docker 容器化的大模型“傻瓜式”部署
本地运行大模型（SenseVoice）通常意味着让用户陷入 Python 环境、CUDA 依赖的“配置地狱”。
**巧思在于**：我们在后台实现了全自动的 Docker 生命周期管理！
只要用户电脑装有 Docker Desktop，VTT 会自动生成 `Dockerfile`、依赖配置并自动在后台 Build 镜像、拉起 Inference 服务容器、监听健康状态。用户只需在 UI 上点一下“下载并启用”，就能获得开箱即用的本地 AI 语音引擎。

### ✂️ 4. 智能键鼠模拟，兼容一切输入框
文本获取后如何输入到目标软件？如果使用传统的剪贴板容易覆盖用户原有的复制内容，或者在某些不支持粘贴的软件里失效。
**巧思在于**：VTT Keyboard 使用 `arboard` 动态写入剪贴板，随后通过 `enigo` 模拟触发底层的 `Ctrl+V` (或 macOS 的 `Cmd+V`) 快捷键。一气呵成，兼容一切支持文本输入的终端或应用。

### 🌍 5. 模块化与 i18n 友好
我们在 React 侧保持了严谨的组件拆分（如 `NumberWheelInput`, `TagInput`, `SegmentedControl` 等精美交互组件），并且所有界面文案均接入了标准的 i18n 国际化管理，从设计之初就面向了全球用户。
